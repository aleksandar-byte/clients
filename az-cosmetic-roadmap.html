The big picture

Building a clearer service system, not just rewriting pages

So far the work has focused on cleaning up confusing service navigation, replacing thin or outdated content, removing old practice and location references, and creating a reusable page system so future service pages can be improved more consistently.

The remaining work is organized by priority. We start with the pages that matter most for patient decisions and local search visibility, then move into supporting service pages and a final sitewide polish.

Reusable page system

A reusable service page structure means new and improved pages follow a consistent pattern, so the site avoids one-off page builds. The goal isn't to make every page identical — it's to keep the underlying structure consistent while letting each page carry the right content, intent, and level of detail.

Technical cleanup

A technical crawl was run and priority issues were addressed, giving the remaining page work a cleaner foundation.

  • Metadata — fixed missing, outdated, or overly long page titles and meta descriptions.
  • Headings & templates — corrected service-template H1 issues that could confuse page structure.
  • Redirects — fixed an incorrect redirect found during crawl review.
  • Link safety — added safer handling for links opening in a new tab.
  • Image dimensions — added key dimensions to reduce layout shift and improve stability.

What we are working toward

The plan ahead, in priority order

1

Final QA for completed pages

Lock down the pages that are already in good shape before moving on. Each gets a full pass across the checks below.

  • Mobile layout
  • Calls to action
  • Menu & internal links
  • Titles & meta descriptions
  • Social sharing tags
  • Schema / structured data
  • Cache & live rendering
2

Move approved draft hubs into the main pages

Several major service pages now have improved draft versions that should become the new main direction. The work is to move those drafts into the main live URLs while keeping them easy to review. The Emergency Dentistry page is now sorted — it’s in the improved list above.

Preventive Dentistry Endodontics / Root Canal Oral Surgery
3

Build the missing Restorative Dentistry hub

Restorative Dentistry is a major category and should be built before rewriting its child pages. It should help patients understand repair and replacement options — fillings, crowns, bridges, dentures, and root canal therapy — so related pages can link back and share consistent language.

4

Improve high-intent standalone service pages

Some pages represent direct patient needs and should read as strong service pages, not just informational content — clearly explaining symptoms, when to schedule, what the visit may involve, and how the practice helps.

Gum Disease Treatment Sleep Apnea Treatment
5

Rewrite child service pages by cluster

Once the major hubs are in place, improve the smaller service pages in grouped batches. This keeps content consistent and makes internal linking cleaner.

Cosmetic — Whitening, Bonding, Veneers, Crowns Restorative — Fillings, Crowns, Bridges, Dentures, Root Canal Preventive — Exams, X-Rays, Fluoride, Mouthguards, Oral Cancer Exam, Extractions Gum / root / oral surgery — Scaling & Root Planing, Gum Recession, Cracked Tooth, Wisdom Teeth, TMJ, Oral Pathology
6

Keep Digital Radiographs as a support page

Digital Radiographs should stay on the site but not act as a main menu service. Remove it from the Services menu if still listed, then link to it from the Dental Technology page where it supports the technology explanation.

7

Strengthen internal linking as each cluster is finished

Links are added as pages are completed, not only at the end. Hub pages link to their most important child services; child pages link back to the parent hub; related pages link to each other where helpful; and blog and informational pages support the main service pages — always pointing toward current, patient-facing services in the cleaned menu.

8

Final Services page & sitewide QA

The Services index is updated last, after the service structure is stable.

  • Services menu
  • Services index
  • Hub & child links
  • Mobile layout
  • CTA consistency
  • Metadata & schema
  • Cache clearing
  • Live spot checks

How to think about the roadmap

A clearer service system, built to keep improving

The work isn't just about rewriting pages — it's about building a system that helps patients now and stays easy to improve over time.

  • 01Patients should understand what each service is for.
  • 02Important treatment categories should feel trustworthy and complete.
  • 03Smaller pages should support the main service pages instead of competing with them.
  • 04The menu should stay clean and practical.
  • 05The site should use consistent calls to action and local practice details.
